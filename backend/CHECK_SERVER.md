# 서버 재시작 확인 가이드

## 문제: Health Check는 되는데 health-check는 안 됨

이 문제는 **서버가 코드 변경사항을 반영하지 못했을 때** 발생합니다.

## 해결 방법

### 1단계: 서버 완전히 중지

터미널에서 실행 중인 서버를 **완전히 중지**하세요:
- `Ctrl + C`를 누르세요
- 여러 번 눌러서 확실히 중지하세요
- "터미널이 종료되었습니다" 같은 메시지가 나올 때까지 기다리세요

### 2단계: 서버 재시작

**새로운 터미널 창**을 열고:

```bash
cd backend
npm run dev
```

서버가 시작되면:
```
서버가 포트 3000에서 실행 중입니다.
라우트 등록 완료: /api/inspection
```

이 메시지들이 보여야 합니다.

### 3단계: 확인

브라우저에서 순서대로 확인:

1. 기본 Health Check:
   ```
   http://localhost:3000/api/health
   ```
   → `{"status":"ok"}` 응답이 와야 함

2. 진단 엔드포인트:
   ```
   http://localhost:3000/api/inspection/health-check
   ```
   → 이제 작동해야 함!

## 여전히 안 되는 경우

### 파일 저장 확인

1. `backend/src/routes/inspection.ts` 파일이 저장되었는지 확인
2. 파일의 19번째 줄에 `router.get('/health-check', ...` 가 있는지 확인

### 서버 로그 확인

서버 시작 시 콘솔에 다음이 출력되는지 확인:
```
서버가 포트 3000에서 실행 중입니다.
라우트 등록 완료: /api/inspection
```

만약 "라우트 등록 완료" 메시지가 없다면 서버 파일에 문제가 있을 수 있습니다.

### 포트 충돌 확인

다른 프로세스가 포트 3000을 사용 중일 수 있습니다:

```bash
# Windows에서
netstat -ano | findstr :3000
```

사용 중이면 해당 프로세스를 종료하세요.
