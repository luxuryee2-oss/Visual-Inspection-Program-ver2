# Azure 권한 요청 및 설정 정보

전산팀에 제공할 요청사항입니다.

## 요청사항

Azure Portal에서 앱 등록을 생성하고, SharePoint에 접근할 수 있는 권한을 설정해주세요.

## 필요한 정보

다음 정보들을 복사해서 제공해주시면 됩니다:

### 1. Azure AD 앱 등록 정보

앱 등록 생성 후 다음 정보를 제공해주세요:

```
1. 애플리케이션(클라이언트) ID: [여기에 복사]
2. 디렉터리(테넌트) ID: [여기에 복사]
3. 클라이언트 시크릿 값: [여기에 복사]
```

### 2. SharePoint 사이트 정보

```
1. SharePoint 사이트 URL: [예: https://yourcompany.sharepoint.com/sites/YourSiteName]
2. 저장할 폴더 경로: InspectionData (기본값)
```

## 설정 방법 (전산팀용)

### 1단계: 앱 등록 생성
1. Azure Portal → Azure Active Directory → 앱 등록
2. "새 등록" 클릭
3. 이름: "Visual Inspection Program"
4. 지원되는 계정 유형: "이 조직 디렉터리에만 있는 계정"
5. 등록 클릭

### 2단계: API 권한 설정
1. 생성된 앱 등록 → API 권한
2. "권한 추가" → "Microsoft Graph" 선택
3. "애플리케이션 권한" 탭 선택
4. `Sites.ReadWrite.All` 검색 후 선택
5. "권한 추가" 클릭
6. "관리자 동의 부여" 클릭하여 승인

### 3단계: 클라이언트 시크릿 생성
1. 앱 등록 → 인증서 및 비밀
2. "새 클라이언트 비밀" 클릭
3. 설명: "SharePoint Access Secret"
4. 만료: 24개월
5. 추가 클릭
6. **중요**: "값" 열의 시크릿을 즉시 복사 (다시 볼 수 없음)

### 4단계: 정보 확인
1. 앱 등록 개요 페이지에서:
   - 애플리케이션(클라이언트) ID 복사
   - 디렉터리(테넌트) ID 복사
2. 인증서 및 비밀 페이지에서:
   - 생성한 클라이언트 시크릿 값 복사

## 참고

- 이 앱 등록은 서버에서 자동으로 SharePoint에 접근하기 위해 사용됩니다
- 사용자들은 Microsoft 계정 없이 웹앱을 사용할 수 있습니다
- 앱 등록은 한 번만 설정하면 되며, 이후 모든 사용자가 사용 가능합니다
