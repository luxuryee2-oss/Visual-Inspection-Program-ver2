## 다음 단계

### 1단계: 리스트 정보 확인

브라우저에서 리스트 정보를 확인하세요:

```
http://localhost:3000/api/inspection/list-info
```

이 URL에서 다음을 확인할 수 있습니다:
- 리스트 ID
- 리스트 이름
- **사용 가능한 모든 컬럼 목록** ← 중요!
- 각 컬럼의 이름, 타입, 필수 여부

### 2단계: 실제 저장 테스트

#### 프론트엔드 실행

다른 터미널에서:

```bash
cd frontend
npm run dev
```

#### 저장 테스트

1. 브라우저에서 프론트엔드 접속
2. 검사 데이터 입력:
   - 제품명 (스캔 또는 입력)
   - 검사자 이름
   - 비고 (선택)
   - 사진 4장 (선택)
3. 저장 버튼 클릭

### 3단계: 서버 콘솔 확인

저장 시도 시 **서버 콘솔**에 상세한 로그가 출력됩니다:

```
============================================================
📥 새로운 저장 요청 수신
============================================================
요청 데이터: {...}
✅ 유효성 검사 통과
🔄 SharePoint 저장 시작...
🔐 SharePoint 저장 함수 시작
🔑 Azure 액세스 토큰 가져오는 중...
✅ 액세스 토큰 획득 성공
사이트 ID 자동 조회 중...
드라이브 ID 자동 조회 중...
리스트 ID 조회 중...
리스트 저장 시작...
✅ 저장 완료!
============================================================
```

## 예상 결과

### 성공 시
- 서버 콘솔: "✅ 저장 완료!"
- 브라우저: "저장되었습니다." 메시지
- SharePoint: 문서 라이브러리와 리스트에 데이터 저장됨

### 실패 시
서버 콘솔에 상세한 오류 메시지가 표시됩니다.

## 확인 사항

저장 시도 후:

1. **서버 콘솔에 어떤 메시지가 출력되나요?**
   - 전체 로그를 복사해서 알려주세요

2. **브라우저에 어떤 메시지가 표시되나요?**
   - 성공 메시지? 오류 메시지?

3. **리스트 정보 확인 결과** (`/api/inspection/list-info`)
   - 리스트 컬럼 목록을 확인해주세요

리스트 정보와 저장 테스트 결과를 알려주세요!
